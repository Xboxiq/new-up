# Inspiration Sources — how to mine them without copying

## bestdesignsonx.com (curated design gallery from X/Twitter)
A feed of high-craft UI shots. Value = TASTE CALIBRATION, not assets.

How to use:
1. Scan for COMPOSITION patterns: where the eye lands first, figure/ground, whitespace ratio.
2. Note TYPOGRAPHY pairings and scale jumps (display vs body contrast).
3. Note SURFACE language: shadow softness, border weight, radius rhythm.
4. Extract the PRINCIPLE, re-express in `--f-*`. Never lift a layout 1:1.
5. Ask: "Would this survive in a calm civic broadsheet?" If it is loud, take only its discipline.

## colorion.co family (kinetics + gradientbuttons)
Generators — copy the TECHNIQUE, then re-color to brand and re-time to `--f-ease-*`.

## backgrounds.supply / grainient.supply
Texture library — recreate grain/mesh in code (see backgrounds.md), do not ship their PNGs.

## github.com/docker/awesome-compose
Not a design resource (Docker Compose samples). Excluded from the visual system.
Listed here only so the exclusion is explicit and intentional.

## The taste test (apply before shipping any borrowed idea)
1. Does it raise CLARITY or only decoration? (decoration-only → cut)
2. Is it on BRAND palette? (purple/rainbow → cut)
3. Does it survive reduced-motion + dark + RTL + 375px?
4. Does `impeccable detect` stay at `[]`?
5. Is it the ONLY signature on this screen? (if not → cut the weaker one)
